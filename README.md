# Фильтр ChoosingBestOffer в Keitaro предназначен для автоматического выбора лучшего лендинга и/или оффера в потоке в зависимости от их эффективности.
Фильтр был доработан с учетом потребностей нашей команды на базе оригинального решения.

Оригинальная версия проекта доступна по ссылке:  
[keitaro-php-filter-choosing-best-offer](https://github.com/ap-etolegko/keitaro-php-filter-choosing-best-offer/tree/main)

Выражаем благодарность основному автору за проделанную работу и предоставленную основу для дальнейшего развития решения.

## 1. С чем работать

Определяет, какие сущности участвуют в анализе и перераспределении трафика:

- **Лендинги**
- **Офферы**
- **Лендинг + оффер** (для схемы «лендинг → оффер»)

> Фильтр перераспределяет трафик только между теми сущностями, которые включены в выбранной схеме.
> В распределении участвуют только активные сущности в схеме

---

## 2. Какую метрику анализировать

Выбирается метрика, по которой определяется лучшая сущность:

- **CR** — конверсия
- **CRs** — подтвержденная конверсия
- **CPA** — стоимость действия
- **% аппрува** — процент подтверждения
- **EPC** — доход на клик
- **ROI** — возврат инвестиций

> Лучшая сущность определяется на основе выбранной метрики за указанный период и в рамках выбранного охвата статистики.

---

## 3. Охват статистики

Определяет уровень анализа данных:

- **Общая по команде**  
  Анализ всей статистики по всем байерам выбранных сущностей.

- **Текущая кампания**  
  Анализ только в рамках текущей кампании `campaign_id`.

- **Наименование объявления FB**  
  Анализ по параметру `creative_id` - Наименование креатива (объявления Facebook).

- **Кампания клика**  
  Анализ по параметру `sub_id_N` — ID кампании Facebook.

- **Адсет клика**  
  Анализ по параметру `sub_id_M` — ID адсета Facebook.

- **Объявление клика**  
  Анализ по параметру `sub_id_K` — ID объявления Facebook.

> `sub_id_N`, `sub_id_M`, `sub_id_K` - номера sub_id задаются в константах фильтра

### Что означает анализ по параметру

Берется параметр текущего клика и выбирается лучшая сущность **только среди кликов с таким же параметром**.

Таким образом можно настроить выбор лучшей сущности отдельно для:
- конкретных кампаний,
- конкретных адсетов,
- конкретных объявлений,
- конкретных креативов.

---

## 4. Минимум конверсий для выбора лучшего

Диапазон: **от 0 до 100 конверсий**.

Определяет минимальное количество конверсий, при котором сущность допускается к участию в выборе лучшей.

> Если количество конверсий в рамках выбранного охвата статистики меньше установленного значения, такая сущность исключается из анализа и не участвует в определении лидера.

---

## 5. Дни статистики

Диапазон: **от 1 до 7 дней**.

Определяет, за какой период учитывать данные:

- `1` — только сегодня  
- `2` — вчера и сегодня  
- `N` — последние N дней включая текущий

---

## 6. Раз в N минут делать перераспределение

Диапазон: **от 10 до 1440 минут (1 сутки)**.

Механика работы:

- После определения лучшей сущности она фиксируется на заданный период.
- Повторная проверка и обновление процентов происходит через N минут.

Пример:

- Клик получен в `10:00`.
- Установлено `30 минут`.
- Следующее обновление распределения произойдет в `10:30`.

---

## 7. Процент доли трафика лучшему варианту

Указывается процент, который будет назначен лучшей сущности.

Оставшийся процент распределяется равномерно между остальными включенными сущностями.

### Пример

Выбрано: **6 лендингов**  
Указанный процент: **80%**

После проверки:

- Лучшая сущность → **80%**
- Оставшиеся 5 → по **4%** каждому  
  (20% ÷ 5 = 4%)

> В случае, когда оставшийся процент не делится поровну, последний лендинг заберет остаток. В варианте выше, если будет еще 1 лендинг, распределение будет 80-3-3-3-3-3-5

---

## 8. Минимум кликов для выбора лучшего

Диапазон: **от 10 до 1000 кликов**.

Определяет минимальное количество кликов, при котором сущность допускается к участию в выборе лучшей.

> Если количество кликов в рамках выбранного охвата статистики меньше установленного значения, такая сущность исключается из анализа и не участвует в определении лидера.
